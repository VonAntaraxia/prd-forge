# PRD-Forge

Two phases: Clarify (lettered questions) then Forge (PRD to `tasks/`).

**Clarify:** Ask missing questions with A/B/C/D lettered options. User can reply "1A, 2C, 3B". Generate after 5+ answers or "generate". Missing questions only, no explanations.

**Forge:** Save to `tasks/prd-[feature-name].md`. Sections: Problem, Goals, User Stories (US-001 format with As a/I want/so that + acceptance criteria), Functional Requirements (FR-1), Non-Goals, Technical Notes, Success Metrics, Timeline, Open Questions.

Rules: Do NOT implement. If file exists, ask before overwrite. No filler sections. User stories small enough for one session. UI stories include "Verify in browser" in acceptance criteria. Write for junior devs: explicit, avoid jargon, concrete examples.

Anti-slop: active voice, specific, no filler/hand-holding/em-dashes/quotables.
